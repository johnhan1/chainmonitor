# PRD v1.0 拆分：任务规划与结果校验点

基于 `PRD-v1.0.md`（自进化链上 Alpha 监控系统，SOL / BASE / BSC / ETH）整理。

## 一、任务规划

### 1. 规划总原则

- 目标导向：先完成“数据 -> 评分 -> 验证 -> 进化”的最小闭环，再扩链、扩因子、扩容量。
- 规划原则：每个任务都绑定可交付物、负责人、截止时间、量化验收口径。
- 执行节奏：按 `Phase 1~4` 推进，采用周迭代、小步发布、每周复盘调参。

### 2. Phase 0（1周）：项目落地准备

- 任务：
  - 明确范围边界（SOL/BASE/BSC/ETH、现货、DEX池与新发行资产）。
  - 建立环境与密钥管理规范（API Key 加密、最小权限、审计日志）。
  - 搭建基础监控告警框架与指标口径。
- 交付物：
  - 架构图 v1（六层架构 + 核心服务边界）。
  - 数据字典 v1（地址、时间轴、token 元数据、实体去重规则）。
  - 指标口径文档（净收益、PF、E、回撤、延迟、缺失率）。
  - 里程碑排期（看板或甘特图）。
- 完成定义：
  - FR/NFR 全部映射到负责人与里程碑。
  - 核心监控项可观测（链路健康、源可用性、延迟、缺失率）。

### 3. Phase 1（2~4周）：多链接入 + 基础打分

- 任务：
  - 实现多链 Ingestion（并行采集、重试、限流、降级）。
  - 实现标准化层（地址格式、统一时间轴、metadata 统一、实体去重）。
  - 落地基础因子与硬过滤 Gate（可交易性、安全风险、反操纵基础规则）。
  - 实现 Scoring Engine：`FinalScore`、`Conviction`、A/B/C 候选池分层。
  - 建立数据源容灾（主备切换、冲突融合）。
  - 建设基础可观测性（日志、指标、告警、数据质量看板）。
- 交付物：
  - 实时信号与候选池看板。
  - 信号解释页（入选原因 + 风险标签 + 失效触发条件）。
  - 运行手册（故障切换和降级策略）。
- 完成定义：
  - 支持持续产出可解释信号。
  - 任一主数据源故障可自动切换。

### 4. Phase 2（3~5周）：回测引擎 + 成本建模

- 任务：
  - 实现事件驱动回测（区块/分钟粒度重放）。
  - 实现成本建模（gas、滑点、手续费、路由延迟、失败交易概率）。
  - 实现策略规则配置（入场/出场/止损/止盈/失效退出）。
  - 建立防过拟合流程（walk-forward、时间切片交叉验证、参数稳定性检验）。
  - 输出归因分析（因子、链别、时段、市场状态维度）。
- 交付物：
  - 回测任务中心（配置、批量运行、导出）。
  - 标准回测报告模板。
  - 参数搜索结果库与版本记录。
- 完成定义：
  - 任一策略版本均可复现回测结果。
  - 核心指标可稳定产出：净收益、PF、E、回撤、容量冲击。

### 5. Phase 3（2~4周）：模拟实盘 + 进化引擎

- 任务：
  - 落地 Paper Trader（实时同口径输入、订单级执行日志、偏差记录）。
  - 落地组合管理（仓位上限、单币预算、链别预算、相关性约束）。
  - 实现 Evolution Engine（周期调权、调阈值、淘汰失效因子）。
  - 实现版本灰度与回滚机制（触发回撤阈值自动回滚）。
- 交付物：
  - 模拟实盘看板与对账报表。
  - 策略版本 A/B 对比报告。
  - 灰度发布与回滚运行手册。
- 完成定义：
  - 形成“回测 -> 模拟 -> 发布 -> 监控 -> 回滚”完整闭环。

### 6. Phase 4（持续）：专项优化与规模化

- 任务：
  - 链别独立参数优化与风险阈值优化。
  - 因子半衰期监控与自动降权。
  - 数据源成本优化（核心因子付费兜底、非核心因子免费优先）。
  - 容量与稳定性压测（峰值流量、故障注入、恢复演练）。
- 交付物：
  - 月度策略版本评估。
  - 成本收益分析报告。
  - 稳定性专项报告。
- 完成定义：
  - 收益、回撤、容量与成本均满足运营目标。

## 二、结果校验点（Gate）

### Gate 1（Phase 1 结束）

- 延迟达标：从异动到信号产出延迟满足链别 SLO（10~60s 级）。
- 稳定性达标：多源容灾后数据缺失率低于阈值。
- 信号可解释：每个信号均有“入选原因 + 风险标签 + 失效触发”。
- 功能达标：A/B/C 候选池正常分层、更新、追踪。

### Gate 2（Phase 2 结束）

- 回测可复现：同一策略版本重复运行结果偏差在可接受范围。
- 成本一致性：成本模型完整纳入并可追溯。
- 抗过拟合：至少完成 1 轮 walk-forward + 参数稳定性验证。
- 策略门槛：MVP 阶段 `PF > 1.2` 且 `E > 0`。

### Gate 3（Phase 3 结束）

- 口径一致：模拟实盘与回测指标口径一致，偏差可量化解释。
- 风控有效：组合约束策略有效阻断超预算风险暴露。
- 版本治理：灰度发布、自动回滚流程可用并经过演练。
- 闭环成立：验证结果能反向驱动参数更新并再次验证。

### MVP 终验（阶段验收）

- 盈利目标：滚动样本组合净收益（扣成本）持续为正。
- 风险收益：`PF > 1.3`（MVP 最低可先达 `>1.2` 后继续优化）。
- 回撤控制：最大回撤在预算内。
- 可用性：核心链路可用性达到 `99.5%+`。
- 可追溯性：任意评分可回放到原始因子快照。
- 容灾能力：核心数据源故障可自动切换并持续服务。

- 周会：收益归因 + 失败样本复盘 + 下周调参计划。
