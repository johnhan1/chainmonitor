# PRD：自进化链上 Alpha 监控系统（SOL / BASE / BSC / ETH）

文档版本

- 版本： v1.0
- 日期： 2026-04-18
- 产品形态：研究与交易决策支持系统（先做“发现+验证+迭代”，后接自动执行）
  一、产品目标
- 目标：在 SOL / BASE / BSC / ETH 上，尽早发现高赔率代币机会，并通过持续验证实现“总体盈利最大化”。
- 核心原则：不追求高胜率，追求正期望与资金曲线向上。
- 方法论： 选币逻辑 是核心， 分析 / 回测 / 模拟实盘 / 调参 全部服务于验证与进化选币逻辑。
- 成功标准：在滚动周期内， 净收益、利润因子、回撤控制、容量稳定性 达标。
  二、业务指标（按优先级）
- P0-总体盈利 ：组合净收益（扣除 gas、滑点、手续费）持续为正。
- P0-风险收益比 ：利润因子 PF > 1.3 （MVP 阶段可先 >1.2 ），最大回撤受控。
- P0-期望值 ：每笔期望收益 E = win\_rate*avg\_win - lose\_rate*avg\_loss > 0 。
- P1-机会捕捉速度 ：从代币上线/异动到信号产出延迟低于设定阈值（链别不同）。
- P1-稳定性 ：多数据源容灾后，数据缺失率低于阈值。
- P2-可解释性 ：每个信号可解释（为什么入选、风险点是什么、失效原因是什么）。
  三、范围定义
- In Scope：链上新币与活跃币监控、实时打分、候选池管理、回测、模拟实盘、自进化参数更新。
- Out of Scope（阶段一）：自动下单实盘、杠杆与衍生品、复杂跨 CEX 套利。
- 交易对象：现货代币优先，重点覆盖 DEX 流动性池与新发行资产。

## 系统级架构

四、总体架构（六层）

- 数据接入层 ：多链节点、区块/交易、池子、持仓、标签、合约安全、社交与舆情。
- 标准化层 ：统一地址格式、统一时间轴、统一 token 元数据、链上实体去重。
- 特征工程层 ：生成选币指标（流动性、资金流、聪明钱、持仓结构、风险因子等）。
- 策略与评分层 ：硬过滤 + 多因子评分 + 风险惩罚 + 置信度估计。
- 验证层 ：历史回测 + 纸面模拟实盘 + 归因分析。
- 进化层 ：基于验证结果自动调权、调阈值、淘汰失效因子，形成新版本策略。
  五、关键服务拆分
- Ingestion Service ：按链并行采集，支持重试、限流、降级。
- Feature Store ：存储实时与历史因子快照，支持回测可复现。
- Scoring Engine ：实时计算 token 分数与信号等级。
- Backtest Engine ：事件驱动回测，支持链别交易成本模型。
- Paper Trader ：模拟成交、滑点、gas、延迟、黑天鹅冲击。
- Evolution Engine ：按周期进行参数搜索、版本评估与灰度发布。
- Observability ：日志、指标、告警、数据质量看板。

## 核心：代币筛选设计

六、数据源策略（免费优先，按成本升序兜底）

- 第1层（免费主用） ：官方/公链 RPC（自建或公共）、 Etherscan/BscScan/BaseScan/SolanaFM 免费接口、 DexScreener 、 GeckoTerminal 、 CoinGecko 免费层、 Dune 公共查询、开源标签库。
- 第2层（低成本补强） ： Covalent 、 Moralis 、 Alchemy/Infura 付费低档、第三方安全扫描 API 低配版。
- 第3层（高可靠兜底） ：商业级链上数据聚合与钱包标签服务（仅在关键数据断供时启用）。
- 兜底机制 ：同一指标至少 2 个来源；主源失败自动切备用；若冲突则按“时间新鲜度+历史准确率”加权融合。
- 成本控制 ：按因子重要度分配预算，核心因子可用付费兜底，非核心因子只用免费源。
  七、选币指标体系（建议 v1）
- 基础可交易性 ：是否在主流 DEX 有池、最小流动性阈值、24h 成交额阈值、池龄。
- 安全与合约风险 ：是否可 mint/黑名单/高税/可升级高风险、权限集中度、审计与已知漏洞标签。
- 流动性质量 ：LP 深度、深度变化率、LP 持有集中度、异常抽池概率。
- 资金行为 ：净流入、主动买盘比例、大额地址净买入、跨时窗资金持续性。
- 持仓结构 ：TopN 持仓占比、新增持有人增长、鲸鱼地址占比变化、老持有人留存。
- 聪明钱信号 ：历史高胜率地址是否建仓、建仓密度、跟随资金扩散速度。
- 价格结构 ：突破形态、回撤幅度、波动率状态、成交量价格共振。
- 叙事热度 ：社媒提及增速、开发活动、跨链叙事相关性（可作为弱因子，防止噪音主导）。
- 反操纵因子 ：刷量检测、同源地址对敲、短时异常成交分布、蜜罐/限制交易风险。
  八、筛选流程（硬过滤 + 打分）
- 硬过滤（Gate） ：先剔除不可交易/高风险/明显操纵标的。
- 基础分（Alpha Score） ：对通过 Gate 的标的做多因子打分。
- 风险惩罚（Risk Penalty） ：对高波动、高集中、合约风险增加惩罚项。
- 置信度（Confidence） ：按数据完整度、跨源一致性、链上信号持续性估计。
- 输出信号 ： A/B/C 三级候选池，A 级进入重点观察与模拟实盘。
  九、评分模型（可落地公式）
- 建议公式：

```
FinalScore = 0.55*AlphaScore + 0.20*MomentumScore + 0.15*SmartMoneyScore
             + 0.10*NarrativeScore - RiskPenalty

Conviction = FinalScore * DataConfidence * RegimeFit
```

- AlphaScore ：流动性、资金流、持仓结构、安全因子的加权和。
- RiskPenalty ：合约风险、持仓过度集中、异常波动、可疑交易行为惩罚。
- DataConfidence ：数据新鲜度、字段完整率、跨源一致性。
- RegimeFit ：市场状态匹配度（风险偏好高/低时使用不同阈值）。
- 权重更新：每周小调、每月大调；依据回测与模拟实盘归因结果自动建议。

## 验证闭环（分析 / 回测 / 模拟 / 调整）

十、分析模块

- 信号解释 ：每个 token 输出“入选原因 + 风险标签 + 失效触发条件”。
- 归因分析 ：收益拆分为因子贡献、链别贡献、时段贡献、市场状态贡献。
- 失败样本库 ：记录亏损信号，标注是“噪音、操纵、流动性坍塌、追高回撤”等原因。
  十一、回测模块
- 回测引擎 ：事件驱动，按区块/分钟粒度重放。
- 成本建模 ：gas、滑点、手续费、路由延迟、失败交易概率。
- 策略规则 ：入场（分数阈值+确认条件）、出场（止损/止盈/时间止盈/失效退出）。
- 评估指标 ：净收益、利润因子、期望值、最大回撤、Calmar、Tail Risk、容量冲击。
- 防过拟合 ：滚动窗口 Walk-forward、时间切片交叉验证、参数稳定性检验。
  十二、模拟实盘模块
- 实时输入 ：接入和线上一致的数据流，避免“回测特供数据”。
- 执行仿真 ：按实时深度估计可成交价，不允许理想成交。
- 组合管理 ：仓位上限、单币风险预算、链别风险预算、相关性约束。
- 对账输出 ：每笔模拟订单有完整成交日志、预估与实际偏差记录。
  十三、自进化模块
- 进化目标 ：最大化滚动净收益与风险调整收益，而非提升胜率。
- 进化对象 ：因子权重、阈值、止盈止损参数、链别参数。
- 进化方法 ：贝叶斯优化 / 网格搜索（MVP 可先网格）、多版本并行评分。
- 发布策略 ：离线优胜后进入小流量灰度，观察稳定后全量。
- 回滚机制 ：新版本触发回撤阈值时自动回滚旧版本。

## 功能需求（FR）

十四、核心功能清单

- FR-001 ：支持 SOL/BASE/BSC/ETH 多链并行采集与统一标准化。
- FR-002 ：支持多数据源按成本阶梯兜底与自动故障切换。
- FR-003 ：提供可配置硬过滤规则（黑名单、风险阈值、流动性阈值）。
- FR-004 ：实时计算并输出 FinalScore 、 Conviction 、风险标签。
- FR-005 ：支持候选池分层管理（A/B/C）与变动追踪。
- FR-006 ：支持回测任务配置、批量运行、报告导出与复现。
- FR-007 ：支持模拟实盘与订单级执行日志。
- FR-008 ：支持策略版本管理、A/B 对比、灰度与回滚。
- FR-009 ：提供归因看板与失败样本分析。
- FR-010 ：提供告警系统（高分新币、风险突变、数据源故障）。
  十五、非功能需求（NFR）
- NFR-001 ：可用性 99.5%+ （核心链路）。
- NFR-002 ：实时信号延迟目标（按链定义 SLO，如 10\~60s 级）。
- NFR-003 ：数据可追溯，任何评分可回放到原始因子快照。
- NFR-004 ：安全合规，API Key 加密、最小权限、审计日志。
- NFR-005 ：成本可控，按月预算上限自动降级非核心数据源。

## 数据与技术建议

十六、数据存储建议

- OLTP ： PostgreSQL （配置、策略版本、任务元数据）。
- 时序/分析 ： ClickHouse （高频指标、回测结果、明细归因）。
- 缓存 ： Redis （实时分数、候选池快照、告警队列）。
- 对象存储 ：回测报告、原始快照归档。
  十七、技术栈建议（MVP）
- 采集与策略 ：Python（生态快，量化库丰富）或 Rust（性能优先）。
- 调度 ：Airflow/Prefect（批任务）+ Kafka/Redpanda（实时流）。
- 可视化 ：Metabase/Grafana + 自建前端（后续）。
- 部署 ：容器化，按链独立 worker，支持水平扩展。

## 里程碑与验收

十八、阶段规划

- Phase 1（2\~4周） ：多链接入、基础因子、硬过滤、初版打分、基础看板。
- Phase 2（3\~5周） ：回测引擎、成本建模、参数搜索、报告体系。
- Phase 3（2\~4周） ：模拟实盘、归因、进化引擎、灰度发布。
- Phase 4（持续） ：链别专项优化、数据源优化、策略组合化。
  十九、MVP 验收标准
- 连续运行稳定，核心数据源故障可自动切换。
- 选币信号可解释且可回放。
- 回测与模拟实盘口径一致，偏差可量化。
- 滚动样本中组合净收益为正，利润因子达标，回撤在预算内。
- 形成“数据→评分→验证→进化→再验证”的闭环流程。

## 风险与应对

二十、主要风险

- 数据污染 ：刷量、机器人、假钱包标签导致误判。
- 过拟合 ：历史有效、未来失效。
- 执行偏差 ：回测成交价过于理想。
- 市场突变 ：叙事切换快，旧因子衰减。
- 单链拥堵 ：时延增大导致信号失真。
  二十一、缓解策略
- 多源交叉验证 + 异常检测 + 可疑样本降权。
- Walk-forward + 稳定性约束，拒绝高方差参数。
- 模拟实盘严格引入滑点与延迟。
- 因子半衰期监控，失效自动降权。
- 链别独立参数与风控阈值，不做“一刀切”。
